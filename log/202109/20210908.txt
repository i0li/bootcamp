【メモ】　
＝＝　GitHub-flowの６つのルール　＝＝
１）masterブランチは常にデプロイ可能である
２）作業用ブランチをmasterから作成する → 何をするブランチなのかが分かるように名前をつける
３）作業用ブランチを定期的にプッシュする
４）プルリクエストを活用する
５）プルリクエストが承認されたらmasterへマージする
６）masterへのマージが完了したら直ちにデプロイを行う

＝＝　shell　＝＝
● for文はseqを用いてコードする
for i in $(seq 1 $END)
do 
　echo $i
done

● 他のスクリプトを呼び出す方法
パスの先頭に「.」「source」「zsh」のどれかをつけることで呼び出せる

● 環境変数は大文字

● 一時変数は使わない

● &&は使わない

● 二択のif文の際はif-elseではなくif-ifを使う　→ ガード節

〈犬かそれ以外の動物かで条件分岐する時〉
----- ガード節 -----
if 犬 ; then 
 処理１
fi
if 犬以外の動物 :then
 処理２
fi
----- ガード節でない -----
if 犬 ; then
  処理1
else ;
　処理２
fi
